# フロントエンドデザイン品質ガイドライン

各デザイン案の作成時、以下のガイドラインに従い「AIっぽい無難なデザイン」を徹底的に回避する。

## デザイン思考

コーディング前に、コンテキストを理解し**大胆な**美的方向性にコミットする:
- **目的**: このインターフェースはどんな問題を解決するか？誰が使うか？
- **トーン**: 極端な方向性を選ぶ: ブルータルミニマル、マキシマリストカオス、レトロフューチャー、オーガニック/ナチュラル、ラグジュアリー/洗練、プレイフル/トイライク、エディトリアル/マガジン、ブルータリスト/ロウ、アールデコ/ジオメトリック、ソフト/パステル、インダストリアル/ユーティリタリアンなど。
- **差別化**: 何がこのデザインを**忘れられない**ものにするか？誰かが覚えているたった一つのことは何か？

**重要**: 明確なコンセプト方向を選び、精密に実行する。大胆なマキシマリズムも洗練されたミニマリズムも有効 — 重要なのは意図的であること。

## フロントエンド美学ガイドライン

- **タイポグラフィ**: 美しく、ユニークで、興味深いフォントを選ぶ。特徴的なディスプレイフォントと洗練されたボディフォントを組み合わせる。
- **カラー＆テーマ**: 一貫した美学にコミットする。CSSカスタムプロパティで一貫性を保つ。支配的な色とシャープなアクセントが、控えめで均等に分散されたパレットに勝る。
- **モーション**: エフェクトとマイクロインタラクションにアニメーションを使う。CSS onlyソリューションを優先。ページロード時のスタガード表示（animation-delay）が、散発的なマイクロインタラクションより効果的。スクロールトリガーやサプライズのあるホバーステートを活用。
- **空間構成**: 予想外のレイアウト。非対称。オーバーラップ。対角線的なフロー。グリッドを破る要素。豊かなネガティブスペースまたは制御された密度。
- **背景＆ビジュアルディテール**: ソリッドカラーをデフォルトにせず、雰囲気と奥行きを作る。グラデーションメッシュ、ノイズテクスチャ、幾何学パターン、レイヤードトランスペアレンシー、ドラマチックなシャドウ、装飾的ボーダー、グレインオーバーレイなどを適用。

## 絶対に使ってはいけないもの

| カテゴリ | 禁止事項 |
|---------|---------|
| フォント | Inter, Roboto, Arial, system-ui などの汎用フォント |
| カラー | 白背景に紫グラデーション、ありがちなAIカラースキーム |
| レイアウト | 予測可能なレイアウトとコンポーネントパターン |
| 全般 | コンテキスト固有の個性を欠くテンプレート的デザイン |

**各案は異なるテーマ・フォント・美学を持つこと。** 同じフォント（例: Space Grotesk）を複数案で使わない。ライト/ダークテーマも案ごとに変える。

## 実装の複雑さを美学に合わせる

- マキシマリストなデザイン → 豊富なアニメーション・エフェクトの精緻なコード
- ミニマリスト/洗練されたデザイン → 抑制と精密さ、スペーシング・タイポグラフィ・繊細なディテールへの注意

## バグを生みにくいデザインにする

| ルール | 詳細 |
|--------|------|
| **非表示は `display:none` で行う** | `opacity:0` + `pointer-events:none` は「見えないが操作可能な要素」を生む。`display:none` を使えばDOMから完全に消える |
| **ボタンは「表示=操作可能」にする** | 段階的に表示するボタンは、表示タイミングまで `display:none` にする |
| **アニメーションはシンプルに** | フェードイン程度に留める。複雑なアニメーションチェーンはタイミングバグの原因 |
| **状態数を最小にする** | 画面の状態が増えるほどバグが増える。「表示/非表示」の2状態で済むデザインを優先する |
